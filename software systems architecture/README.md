
# Архитектура программных систем
## **О предмете**
Дисциплина освещает вопросы организации процесса разработки программных систем. Цель дисциплины - сформировать у студентов понимание архитектуры программных систем, их жизненного цикла, принципов разработки внешнего программного проекта и проведения технико-экономического анализа проектов программных систем.

Продолжительность: 1 семестр.

## Лабораторные работы

### [**Лабораторная работа №1**](https://github.com/kihort-si/itmo/tree/main/software%20systems%20architecture/lab1)

Выбрать любую реально существующую систему и описать её в терминах UML.
Желательно, чтобы система была не полностью информационной, но опиралась на
информационную систему как показано в примере на лекции (Point of sale). Необходимо
описать границы системы на разных уровнях, а также описать сценарии использования
для нескольких Акторов.

### [**Лабораторная работа №2**](https://github.com/kihort-si/itmo/tree/main/software%20systems%20architecture/lab2)

Из списка шаблонов проектирования GoF и GRASP выбрать 3-4 шаблона и для каждого из
них придумать 2-3 сценария, для решения которых могу применены выбранные шаблоны.
Сделать предположение о возможных ограничениях, к которым можем привести
использование шаблона в каждом описанном случае. Обязательно выбрать шаблоны из
обоих списков.
